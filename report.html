Introduction

This project seeks to discover which position is most reliable when drafting with a lottery pick (first 14 picks of any given draft). It does this by using Omega-style analysis to evaluate historical picks based on the position they played during their rookie year. Omega Ratios above 1 mean a position is more likely to exceed expectations, and the opposite is true for Omega Ratios < 1.

Data & Methodology

Preliminary Information:

  • Data source: Basketball Reference (https://www.basketball-reference.com)
  • Metric: Box Plus-Minus, with any BPM ≤ -5 considered effectively “useless.” All players with a BPM less than -5 have their BPM set to -5 so as not to skew the data.
  • Window of effect: Players are evaluated by the average of their 3rd to 5th years — the period when their ability/inability to lead a team should really begin to shine.
  • Punishment for missing a season: Players who miss a season are assigned a BPM of -5 for that season. Availability is the best ability!
  • Range of each bin: Players are sorted into bins consisting of two picks each (e.g., picks 1–2, 3–4, and so on).
  • Position tagging: Each player is tagged based on the position they played during their rookie year. This ensures we have position data for each player, even if they wash out of the NBA quickly. While this system is not perfect, it is reliable enough to be used for our data.

#Scraping draft info from 2011 to 2021
scrape_draft <- function(year) {
  message("Scraping draft: ", year)
  Sys.sleep(3)
  url <- paste0("https://www.basketball-reference.com/draft/NBA_", year, ".html")
  
  headers <- add_headers(`User-Agent` = "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36")
  
  response <- GET(url, headers)
  
  page <- read_html(httr::content(response, "text", encoding = "UTF-8"))
  
  table <- page |>
    html_element("#stats") |>
    html_table(header = FALSE)
  
  colnames(table) <- make.unique(as.character(table[2, ]))
  
  table |>
    slice(-(1:2)) |>
    mutate(draft_year = year)
}

#Now scraping advanced stats from 2014 to 2026
scrape_advanced <- function(season_end_year) {
  message("Scraping advanced stats: ", season_end_year)
  Sys.sleep(3)
  url <- paste0("https://www.basketball-reference.com/leagues/NBA_", season_end_year, "_advanced.html")

Now, we need to process the data a bit more to extract what we need. This includes finding the what years each player played their 3rd to 5th season, filtering out non-lottery picks, defining what a worthless (<5 BPM) player is, and joining all of this information to our other data frame. Finally, we compute each player’s average BPM across the three year window.

Code 
# Gets players' 3rd to 5th seasons,
# and connects them to their corresponding year
player_avg_bpms <- draft_clean |>
  mutate(
    season_3 = draft_year + 3,
    season_4 = draft_year + 4,
    season_5 = draft_year + 5
  ) |>
  pivot_longer(cols = c(season_3, season_4, season_5),
               names_to = "season_number",
               values_to = "season_year") |>
  # Join player position to data
  left_join(player_positions, by = "player") |>
  left_join(manual_positions |> rename(manual_position = position), by = "player") |>
  mutate(position = coalesce(position, manual_position)) |>
  select(-manual_position) |>
  # ONLY look at lottery picks
  filter(pick <= 14) |>
  # Join to advanced stats
  left_join(advanced_clean, by = c("player" = "player", "season_year" = "season")) |>
  # If the player didn't play, they are considered "worthless" (BPM = -5)
  # Players cannot be less than "worthless" (BPM < -5)
  mutate(BPM = if_else(is.na(BPM), -5, BPM),
         BPM = if_else(BPM < -5, -5, BPM)) |>
  # Compute average BPM across three year window
  group_by(player, pick, draft_year, position) |>
  summarise(avg_BPM = mean(BPM), .groups = "drop")

Building the Metric

Defining “expected” performance

We calculate some basic statistics by “pick-range” (two picks per range) below. This includes getting an average BPM from all players in each pick-range and using this data to conclude expected_BPM, std_deviation, etc. The expected performance is then assumed to be the average BPM by pick-range.

Code 
# Calculate expected BPM + other data by pick range
pick_range_stats <- player_avg_bpms |>
  group_by(player, pick, draft_year) |>
  summarise(avg_BPM = mean(avg_BPM), .groups = "drop") |>
  mutate(pick_range = cut(pick, breaks = seq(0, 60, by = 2),
                          labels = paste(seq(1, 56, by = 2), seq(2, 60, by = 2), sep = "-"))) |>
  filter(!is.na(pick_range)) |>
  group_by(pick_range) |>
  summarise(
    expected_BPM     = mean(avg_BPM),
    std_deviation    = sd(avg_BPM),
    downside_dev     = sqrt(mean(pmin(avg_BPM - mean(avg_BPM), 0)^2)),
    upside_dev      = sqrt(mean(pmax(avg_BPM - mean(avg_BPM), 0)^2)),
    n                = n()
  )

Players are then sorted into pick-ranges.

Code 
# Sort all players by pick range
player_bpm_by_pick_range <- player_avg_bpms |>
  group_by(player, pick, draft_year, position) |>
  summarise(avg_BPM = mean(avg_BPM), .groups = "drop") |>
  mutate(pick_range = cut(pick, breaks = seq(0, 60, by = 2),
                          labels = paste(seq(1, 56, by = 2), seq(2, 60, by = 2), sep = "-"))) |>
  filter(!is.na(pick_range))

First attempt: Sortino-style ratio

My first attempt at quantifying player reliability was to have a sortino-style ratio that would compute the average residual of each position compared to the expected BPM for their pick range, then dividing by its downside deviation. However, this meant that having a negative residual would actually benefit the ratio in question, which is wrong.

The fix: switching to Omega ratio

Switching to an Omega ratio, which divides the total upside of a group by its total downside, fixes my problem.

Code 
# Compute omega score by position,
# using residuals when compared to expected value
omega_scores <- player_bpm_by_pick_range |>
  filter(!is.na(position)) |>
  left_join(pick_range_stats, by = "pick_range") |>
  mutate(residual = avg_BPM - expected_BPM) |>
  group_by(position) |>
  summarise(
    upside_sum   = sum(pmax(residual, 0)),
    downside_sum = abs(sum(pmin(residual, 0))),
    omega_ratio  = if_else(downside_sum == 0, NA_real_, upside_sum / downside_sum),
    n            = n()
  ) |>
  arrange(desc(omega_ratio))

Omega works here because it gives us an idea of downside as well as upside; one position may generate many exceptional stars, but if the average player is less than ideal, perhaps a better option may exist.

Limitations

  • Position-tagging ambiguity, since positional data is taken from rookie years only, the project does not take into account players who developed into a different position than the one they originally played.
  • Small sample sizes due to the inherent size of the draft could cause issues.
  • Scraping reliability: Basketball Reference does not like frequent scraping requests.

Future Work

  • Anchor expected performance to an external reference (consensus mock draft rankings or ADP) rather than only internal pick-range bins, to separate true positional risk from modeling bias.
  • Resolve positional discrepancies.
  • Revisit EPM (Dunks & Threes) as a data source if cost/access constraints change. EPM is generally more reliable than BPM as it does a better job considering context.

## Data & Methodology

**Preliminary Information:**

- **Data source:** Basketball Reference (https://www.basketball-reference.com)
- **Metric:** Box Plus-Minus, with any BPM ≤ -5 considered effectively "useless." All players with a BPM less than -5 have their BPM set to -5 so as not to skew the data.
- **Window of effect:** Players are evaluated by the average of their 3rd to 5th years — the period when their ability/inability to lead a team should really begin to shine.

- **Position tagging:** Each player is tagged based on the position they played during their rookie year. This ensures we have position data for each player, even if they wash out of the NBA quickly. While this system is not perfect, it is reliable enough to be used for our data.

```{r}
  summarise(avg_BPM = mean(BPM), .groups = "drop")
```

## Building the Metric

### Defining "expected" performance

We calculate some basic statistics by "pick-range" (two picks per range) below. This includes getting an average BPM from all players in each pick-range and using this data to conclude expected_BPM, std_deviation, etc. The expected performance is then assumed to be the average BPM by pick-range.

```{r}
#| label: compute-stats-by-pick-range

# Calculate expected BPM + other data by pick range
pick_range_stats <- player_avg_bpms |>
  group_by(player, pick, draft_year) |>
  summarise(avg_BPM = mean(avg_BPM), .groups = "drop") |>
  mutate(pick_range = cut(pick, breaks = seq(0, 60, by = 2),
                          labels = paste(seq(1, 56, by = 2), seq(2, 60, by = 2), sep = "-"))) |>
  filter(!is.na(pick_range)) |>
  group_by(pick_range) |>
  summarise(
    expected_BPM     = mean(avg_BPM),
    std_deviation    = sd(avg_BPM),
    downside_dev     = sqrt(mean(pmin(avg_BPM - mean(avg_BPM), 0)^2)),
    upside_dev      = sqrt(mean(pmax(avg_BPM - mean(avg_BPM), 0)^2)),
    n                = n()
  )
```

Players are then sorted into pick-ranges.

```{r}
#| label: sort_players_by_pick_range

# Sort all players by pick range
player_bpm_by_pick_range <- player_avg_bpms |>
  group_by(player, pick, draft_year, position) |>
  summarise(avg_BPM = mean(avg_BPM), .groups = "drop") |>
  mutate(pick_range = cut(pick, breaks = seq(0, 60, by = 2),
                          labels = paste(seq(1, 56, by = 2), seq(2, 60, by = 2), sep = "-"))) |>
  filter(!is.na(pick_range))
```

### First attempt: Sortino-style ratio

My first attempt at quantifying player reliability was to have a sortino-style ratio that would compute the average residual of each position compared to the expected BPM for their pick range, then dividing by its downside deviation. However, this meant that having a negative residual would actually benefit the ratio in question, which is wrong.

### The fix: switching to Omega ratio

Switching to an Omega ratio, which divides the total upside of a group by its total downside, fixes my problem.

```{r}
#| label: omega-fix

# Compute omega score by position,
# using residuals when compared to expected value
omega_scores <- player_bpm_by_pick_range |>
  filter(!is.na(position)) |>
  left_join(pick_range_stats, by = "pick_range") |>
  mutate(residual = avg_BPM - expected_BPM) |>
  group_by(position) |>
  summarise(
    upside_sum   = sum(pmax(residual, 0)),
    downside_sum = abs(sum(pmin(residual, 0))),
    omega_ratio  = if_else(downside_sum == 0, NA_real_, upside_sum / downside_sum),
    n            = n()
  ) |>
  arrange(desc(omega_ratio))
```

Omega works here because it gives us an idea of downside as well as upside; one position may generate many exceptional stars, but if the average player is less than ideal, perhaps a better option may exist.

## Limitations

- **Position-tagging ambiguity**, since positional data is taken from rookie years only, the project does not take into account players who developed into a different position than the one they originally played.
- **Small sample sizes** due to the inherent size of the draft could cause issues.
- **Scraping reliability**: Basketball Reference does not like frequent scraping requests.

## Future Work

- Anchor expected performance to an external reference (consensus mock draft rankings or ADP) rather than only internal pick-range bins, to separate true positional risk from modeling bias.
- Resolve positional discrepancies.
- Revisit EPM (Dunks & Threes) as a data source if cost/access constraints change. EPM is generally more reliable than BPM as it does a better job considering context.
